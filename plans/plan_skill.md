# New Skill Functions

## Overview

The PyBFS package is used for baseflow separation and forecasting. We wish to add some new functions to systematically assess how well the pybfs performs at both baseflow separation and forecasting. These functions will be located in `pybfs/skill.py`. This document describes both of these new functions. The first is `separation_skill()`, which will assess how well the pybfs baseflow separation matches a reference baseflow. The second function, `forecast_skill()`, assesses how well the pybfs forecast performs during naturally-occurring baseflow-dominated periods. Both functions rely on `modified_strict_baseflow()` to identify baseflow-dominated days. This is a statistical method that uses streamflow derivatives and a quantile threshold to flag days where observed streamflow is likely composed entirely of baseflow — it does not use any pybfs model parameters. On these flagged days, the observed streamflow serves as the reference baseflow against which pybfs outputs are compared.

## Modified Strict Baseflow Function

This will be a new function `modified_strict_baseflow()` in `pybfs/skill.py`. To be consistent with the rest of the pybfs API, it accepts a DataFrame with `Date` and `Streamflow` columns (same as `bfs()`). It returns a boolean array of the same length as the input, where `True` indicates a day identified as baseflow by the modified strict method. This is consistent with the `strict_baseflow` interface from pybaseflow (https://github.com/BYU-Hydroinformatics/pybaseflow).

### Function Signature

```python
def modified_strict_baseflow(hydrograph, quantile=0.8):
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `hydrograph` | `pd.DataFrame` | Observed streamflow with columns `Date` (datetime) and `Streamflow` (m³/day) — same as `bfs()` |
| `quantile` | `float` | Quantile value for the strict baseflow filter (default 0.8) |

### Returns

`np.ndarray` — Boolean array of length `len(hydrograph)`. `True` = baseflow day, `False` = non-baseflow day.

### Reference Implementation

Here is a copy of a function generated by Eniola at the University of Alabama that we will adapt:

```python
def modified_strict_baseflow(streamflow_data_series, dates_series, quantile=0.8):
    """
    Calculates the modified strict baseflow from streamflow data by integrating
    the strict baseflow identification and flow thresholding logic.

    Args:
        streamflow_data_series (np.ndarray): The streamflow time series (Q).
        dates_series (pd.Series): The corresponding date series for the streamflow data.
        quantile (float, optional): The quantile value used in strict_baseflow.
                                    Default is 0.8.

    Returns:
        tuple: A tuple containing:
            - sim (np.ndarray): The modified strict baseflow values.
            - metric_dates (np.ndarray): The dates corresponding to the modified strict baseflow.
    """

    def strict_baseflow_internal(Q, ice=None, quantile=0.8):
        dQ = (Q[2:] - Q[:-2]) / 2

        # 1. flow data associated with positive and zero values of dy / dt
        wet1 = np.concatenate([[True], dQ >= 0, [True]])

        # 2. previous 2 points before points with dy/dt≥0, as well as the next 3 points
        idx_first = np.where(wet1[1:].astype(int) - wet1[:-1].astype(int) == 1)[0] + 1
        idx_last = np.where(wet1[1:].astype(int) - wet1[:-1].astype(int) == -1)[0]
        idx_before = np.repeat([idx_first], 2) - np.tile(range(1, 3), idx_first.shape)
        idx_next = np.repeat([idx_last], 3) + np.tile(range(1, 4), idx_last.shape)
        idx_remove = np.concatenate([idx_before, idx_next])
        wet2 = np.full(Q.shape, False)
        wet2[idx_remove.clip(min=0, max=Q.shape[0] - 1)] = True

        #3
        #  recession limbs in the 80th quantile
        wet3_core = np.concatenate([[True], dQ[1:] - dQ[:-1] < 0, [True, True]])

        # Get indices where the rule is True
        idx3_all = np.where(wet3_core)[0]

        # Filter to only keep indices where flow is in the 90th percentile
        q80 = np.quantile(Q, quantile)
        idx3 = idx3_all[Q[idx3_all] >= q80]

        # Expand: remove 4 points before and 6 after each valid index
        idx3_buffer = np.repeat([idx3], 10) + np.tile(range(-4, 6), idx3.shape)

        # Clip and mark in full-length Boolean mask
        wet3 = np.full(Q.shape, False)
        wet3[idx3_buffer.clip(min=0, max=Q.shape[0] - 1)] = True

        # 4. flow data followed by a data point with a larger value of -dy / dt
        wet4 = np.concatenate([[True], dQ[1:] - dQ[:-1] < 0, [True, True]])

        # dry points, namely strict baseflow
        dry = ~(wet1+wet2+wet3+wet4)

        # avoid ice conditions which invalidate the groundwater-baseflow relationship
        if ice is not None:
            dry[ice] = False

        return dry

    def flow_thresh_internal(Q, strict_mask):
        """
        Return streamflow values where the gradient is positive
        and Q ≤ max(Q[strict_mask]).
        """
        Q = np.asarray(Q)
        Q_strict_avg = np.mean(Q[strict_mask])
        valid_indices = np.where(Q <= Q_strict_avg)[0]
        return valid_indices

    # Apply the strict baseflow identification mask using the internal function
    strict_mask = strict_baseflow_internal(streamflow_data_series, quantile=quantile)

    # Identify valid indices based on flow threshold using the internal function
    valid_indices = flow_thresh_internal(streamflow_data_series, strict_mask)

    # Combine indices from strict baseflow and flow threshold
    strict_indy = np.where(strict_mask)[0]
    combined_indy = np.union1d(valid_indices, strict_indy)

    # Extract the modified strict baseflow values and their corresponding dates
    sim = streamflow_data_series[combined_indy]
    metric_dates = dates_series.iloc[combined_indy].values

    return sim, metric_dates

```

**Changes needed from reference implementation**:
1. Change input from `(streamflow_data_series, dates_series)` to `(hydrograph)` DataFrame with `Date`/`Streamflow` columns — consistent with `bfs()`.
2. Change return from `(sim, metric_dates)` tuple to a single boolean array of length `len(hydrograph)`.
3. Internally extract `Q = np.array(hydrograph['Streamflow'])` and operate on that.
4. Build the combined boolean mask in-place rather than extracting subset values.

## Separation Skill Function

We will implement the `separation_skill()` function, which will evaluate how well the pybfs baseflow separation matches the modified strict baseflow as a reference. This function will compute the pybfs baseflow separation using the `bfs()` function, and then compare it to the modified strict baseflow to compute skill metrics (RMSE and MAE) across all days. The function will return a DataFrame with daily records and overall skill metrics.

### Function Signature

```python
def separation_skill(hydrograph, SBT, basin_char, gw_hyd, flow,
                     quantile=0.8):
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `hydrograph` | `pd.DataFrame` | Observed streamflow with columns `Date` (datetime) and `Streamflow` (m³/day) |
| `SBT` | `pd.DataFrame` | Baseflow table from `base_table()` with columns `['Xb','Z','S','Q']` |
| `basin_char` | `list` | `[area, lb, x1, wb, por]` — same as `bfs()` |
| `gw_hyd` | `list` | `[alpha, beta, ks, kb, kz]` — same as `bfs()` |
| `flow` | `list` | `[qthresh, rs, rb1, rb2, prec, fr4rise]` — same as `bfs()` |
| `quantile` | `float` | Quantile value used in modified strict baseflow (default 0.8) |

### Returns

`(skill_df, metrics)` — a DataFrame and a dictionary as described below.    

### Output DataFrame

| Column | Description |
|--------|-------------|
| `Date` | Datetime of each record |
| `Q` | Observed streamflow (m³/day) |
| `BF_strict` | Boolean baseflow flag from modified strict method (`True` = baseflow day) |
| `BF_bfs` | Baseflow value (m³/day) from `bfs()` separation |
| `RES` | Residual `Q - BF_bfs` on strict-baseflow days; `NaN` otherwise |

> **Design Note**: On days identified as baseflow-dominated by the modified strict method (`BF_strict == True`), the observed streamflow `Q` is treated as the reference baseflow. The residual `Q - BF_bfs` measures how well the pybfs baseflow separation matches this reference. RMSE and MAE are computed only on these strict-baseflow days. Days where `BF_strict == False` have `RES = NaN` and are excluded from metrics.

### Overall Metrics

| Key | Description |
|-----|-------------|
| `RMSE` | Root Mean Square Error of `Q - BF_bfs` across all strict-baseflow days |
| `MAE` | Mean Absolute Error of `Q - BF_bfs` across all strict-baseflow days |
| `n_days` | Number of strict-baseflow days used in the computation |
| `frac_strict` | Fraction of total days identified as strict baseflow |

### Implementation Steps

#### Step 1: Validate Inputs
- Verify `hydrograph` has `Date` and `Streamflow` columns.
- Verify `SBT`, `basin_char`, `gw_hyd`, and `flow` are the correct types and lengths.
- Ensure `0 < quantile < 1`.

#### Step 2: Compute Modified Strict Baseflow Mask
- Call `modified_strict_baseflow(hydrograph, quantile=quantile)` to get a boolean array.
- Handle NaN streamflow values: set corresponding mask entries to `False`.

#### Step 3: Run BFS Baseflow Separation
- Run `bfs(hydrograph, SBT, basin_char, gw_hyd, flow)` on the full hydrograph.
- Extract the `Baseflow` column from the result.

#### Step 4: Build `skill_df`
- Create DataFrame with columns:
  - `Date`: from hydrograph.
  - `Q`: observed streamflow.
  - `BF_strict`: boolean mask from Step 2.
  - `BF_bfs`: baseflow values from Step 3.
  - `RES`: initialized to `NaN`.
- On rows where `BF_strict == True`, compute `RES = Q - BF_bfs`.

#### Step 5: Compute Overall Metrics
- Select rows where `BF_strict == True` and both `Q` and `BF_bfs` are non-NaN.
- Compute:
  - `RMSE = sqrt(mean(RES²))` across selected rows.
  - `MAE = mean(|RES|)` across selected rows.
  - `n_days`: count of selected rows.
  - `frac_strict`: `n_days / total_days`.
- Return as dictionary: `{'RMSE': ..., 'MAE': ..., 'n_days': ..., 'frac_strict': ...}`.

#### Step 6: Return Results
```python
return skill_df, metrics
```

### Sample Usage

```python
import pandas as pd
import numpy as np
import pybfs

# Load streamflow data
streamflow_data = pd.read_csv('docs/files/2312200_data.csv')
streamflow_data['Date'] = pd.to_datetime(streamflow_data['Date'])

# Load site parameters
bfs_params = pd.read_csv('docs/files/bfs_params_50.csv')
site_number = 2312200
basin_char, gw_hyd, flow = pybfs.get_values_for_site(bfs_params, site_number)

# Generate baseflow table
lb, x1, wb, por = basin_char[1], basin_char[2], basin_char[3], basin_char[4]
alpha, beta, ks, kb, kz = gw_hyd
SBT = pybfs.base_table(lb, x1, wb, beta, kb, streamflow_data, por)

# Run separation skill assessment
skill_df, metrics = pybfs.separation_skill(
    streamflow_data, SBT, basin_char, gw_hyd, flow,
    quantile=0.8
)

# Inspect results
print("=== Separation Skill Metrics ===")
print(f"RMSE:          {metrics['RMSE']:.2f} m³/day")
print(f"MAE:           {metrics['MAE']:.2f} m³/day")
print(f"Strict days:   {metrics['n_days']} ({metrics['frac_strict']:.1%} of record)")

# Compare on strict-baseflow days only
strict_days = skill_df[skill_df['BF_strict']]
print(f"\nMean observed Q on strict days:  {strict_days['Q'].mean():.2f}")
print(f"Mean BFS baseflow on strict days: {strict_days['BF_bfs'].mean():.2f}")
```

## Forecast Skill Function

Next, we will implement the `forecast_skill()` function, which will evaluate the skill of the pybfs forecast during naturally-occurring baseflow-dominated periods. This function first calls the modified_strict_baseflow function to identify baseflow-dominated days, then identifies contiguous sequences of such days that meet the minimum length and saturation criteria. For each sequence, it runs a forecast initialized with the observed conditions immediately preceding the sequence, and compares the forecasted baseflow to the observed streamflow days where the baseflow is identified, to compute skill metrics (RMSE and MAE). The function returns a detailed DataFrame with daily records and a summary DataFrame with one row per sequence.

### Function Signature

```python
def forecast_skill(hydrograph, SBT, basin_char, gw_hyd, flow,
                   min_days=30, max_days=90, min_sat=0.9, train_days=365,
                   quantile=0.8):
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `hydrograph` | `pd.DataFrame` | Observed streamflow with columns `Date` (datetime) and `Streamflow` (m³/day) |
| `SBT` | `pd.DataFrame` | Baseflow table from `base_table()` with columns `['Xb','Z','S','Q']` |
| `basin_char` | `list` | `[area, lb, x1, wb, por]` — same as `bfs()` |
| `gw_hyd` | `list` | `[alpha, beta, ks, kb, kz]` — same as `bfs()` |
| `flow` | `list` | `[qthresh, rs, rb1, rb2, prec, fr4rise]` — same as `bfs()` |
| `min_days` | `int` | Minimum number of days a sequence must span to be included (default 30) |
| `max_days` | `int` | Maximum forecast period — sequences longer than this are truncated (default 90) |
| `min_sat` | `float` | Minimum fraction of baseflow days in sequence required to consider a sequence as baseflow-dominated (default 0.9) |
| `train_days` | `int` | Number of days of observed data immediately before a sequence used to initialize the forecast (default 365) |
| `quantile` | `float` | Quantile value used in modified strict baseflow (default 0.8) |

### Returns

`(skill_df, summary_df, metrics)` — two dataframes and a dictionary as described below.

---

### Output DataFrames

#### `skill_df` — daily record for the gage

| Column | Description |
|--------|-------------|
| `Date` | Datetime of each record |
| `Q` | Observed streamflow (m³/day) |
| `BF` | Binary baseflow flag generated by the modified strict method |
| `SEQ` | Sequence number (1-indexed integer) for days inside a valid sequence; `NaN` otherwise |
| `FC` | Forecasted baseflow (m³/day) from `pybfs.forecast()`; `NaN` outside sequences |
| `RES` | Residual `Q - FC`; `NaN` outside sequences |

Note: Each sequence will have a unique integer: 1, 2, 3, ... for all days in that sequence. Days not in any sequence will have `SEQ = NaN`.

#### `summary_df` — one row per sequence

| Column | Description |
|--------|-------------|
| `SEQ` | Sequence number |
| `LEN` | Length of sequence in days (≥ `min_days`) |
| `SAT` | Saturation fraction (fraction of baseflow days in the sequence) |
| `RMSE` | Root Mean Square Error of residuals within the sequence |
| `MAE` | Mean Absolute Error of residuals within the sequence |

#### `metrics` — dictionary of overall metrics

| Key | Description |
|-----|-------------|
| `overall_RMSE` | RMSE computed across all sequence days combined |
| `overall_MAE` | MAE computed across all sequence days combined |

### Implementation Steps

#### Step 1: Validate Inputs
- Verify `hydrograph` has `Date` and `Streamflow` columns.
- Verify `SBT`, `basin_char`, `gw_hyd`, and `flow` are the correct types and lengths.
- Ensure `min_days > 0`, `0 < min_sat <= 1`, `train_days > 0`, `max_days > 0`.

#### Step 2: Compute Modified Strict Baseflow Mask
- Call `modified_strict_baseflow(hydrograph, quantile=quantile)` to get a boolean array of the same length as the hydrograph.
- Handle NaN streamflow values: set corresponding mask entries to `False`.

#### Step 3: Initialize `skill_df`
- Create `skill_df` with columns: `Date`, `Q` (observed streamflow), `BF` (boolean mask from Step 2).
- Initialize `SEQ`, `FC`, and `RES` columns as `NaN`.

#### Step 4: Identify Contiguous Baseflow Sequences
- Find contiguous runs of `True` values in the `BF` column.
- For each run, compute:
  - `length`: total number of days in the run.
  - `saturation`: fraction of `BF == True` days within the run (will be 1.0 for contiguous True runs, but useful if we later relax strict contiguity).
- Filter runs: keep only those with `length >= min_days` and `saturation >= min_sat`.
- Assign sequence numbers (1, 2, 3, ...) to qualifying runs in the `SEQ` column of `skill_df`.

#### Step 5: Forecast Each Sequence
For each qualifying sequence `i`:

1. **Determine training period**: Take `train_days` of data immediately before the sequence start. If fewer days are available (sequence starts near the beginning of the record), use all available prior data.
2. **Run `bfs()`** on the training period hydrograph to get model state:
   ```python
   train_df = hydrograph[train_start:seq_start]
   bfs_result = bfs(train_df, SBT, basin_char, gw_hyd, flow)
   ```
3. **Extract initial conditions** from the last row of the BFS result:
   ```python
   last = bfs_result.iloc[-1]
   ini = (last['X'], last['Zb.L'], last['Zs.L'],
          last['StBase'], last['StSur'],
          last['SurfaceFlow'], last['Baseflow'], last['Rech'])
   ```
4. **Create forecast DataFrame** for the sequence period, capped at `max_days`:
   ```python
   seq_len = min(len(sequence), max_days)
   forecast_dates = pd.date_range(start=seq_start_date, periods=seq_len, freq='D')
   forecast_input = pd.DataFrame({'Date': forecast_dates, 'Streamflow': np.nan})
   ```
   > **Note**: The existing `forecast()` function in `utilities.py` currently expects lowercase `date`/`streamflow` columns. This should be updated to uppercase `Date`/`Streamflow` for consistency with `bfs()` and the rest of the pybfs API.
5. **Run `forecast()`**:
   ```python
   fc_result = forecast(forecast_input, SBT, basin_char, gw_hyd, flow, ini)
   ```
6. **Populate `skill_df`**: Map the forecasted `Baseflow` values from `fc_result` into the `FC` column of `skill_df` for the corresponding dates.
7. **Compute residuals**: `RES = Q - FC` for each day in the sequence.

#### Step 6: Build `summary_df`
For each sequence, compute one row with:
- `SEQ`: Sequence number.
- `LEN`: Length of the sequence in days.
- `SAT`: Saturation fraction (baseflow days / total days in sequence).
- `RMSE`: Root Mean Square Error of `RES` within the sequence (computed on days where both `Q` and `FC` are valid).
- `MAE`: Mean Absolute Error of `RES` within the sequence.

#### Step 7: Compute Overall Metrics
- Pool all residuals from all sequences into a single array.
- Compute `overall_RMSE` and `overall_MAE` across all pooled residuals.
- Return as a dictionary: `{'overall_RMSE': ..., 'overall_MAE': ...}`.

#### Step 8: Return Results
```python
return skill_df, summary_df, metrics
```


### Notes and Edge Cases

- If a sequence starts within the first `train_days` days of the record, use all available prior data (do not skip).
- If `Qob == 0` for a day, skip BFI computation for that day (`BF = 0` by default).
- NaN streamflow values in the hydrograph are treated as non-baseflow days (`BF = 0`).
- The `forecast()` function assumes no precipitation (Impulse = 0), which is appropriate for baseflow-only forecasting during recession periods.
- `FC` represents the forecasted **baseflow** (not total streamflow), so `RES = Q - FC` captures total streamflow minus the baseflow forecast component.
